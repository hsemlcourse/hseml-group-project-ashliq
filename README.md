## Описание задачи

**Задача:** Классификация.

В этом проекте рассматривается задача предсказания коммерческой успешности фильма по его признакам.
Целевая переменная в исходном датасете отсутствует, поэтому она задаётся самостоятельно.

В качестве основы таргета используется показатель:

`ROI = revenue / budget`

После этого фильмы разбиваются на 3 класса:

* `low` — низкая коммерческая успешность;
* `medium` — средняя;
* `high` — высокая.

Такой способ постановки задачи был выбран потому, что ROI лучше отражает относительную окупаемость фильма, чем абсолютная прибыль.

**Датасет:** `MOVIES.csv` с информацией о фильмах.
В ноутбуке используется локальный файл `MOVIES.csv`.

**Целевая метрика:** `macro F1`.

Дополнительно в ноутбуке анализируется `accuracy`.
Основной метрикой выбрана `macro F1`, потому что задача многоклассовая, и важно учитывать качество на всех классах, а не только общую долю правильных ответов.

## Данные

В ноутбуке используется датасет `MOVIES.csv`.

По результатам первичного просмотра данных:

* датасет содержит **44 985 строк** и **23 столбца**;
* в данных есть числовые, категориальные и текстовые признаки;
* в ряде столбцов присутствуют пропуски;
* датасет подходит для постановки задачи классификации коммерческой успешности фильмов.

Примеры используемых признаков:

* числовые: `budget`, `revenue`, `popularity`, `runtime`, `vote_average`, `vote_count`, `year`;
* категориальные: `adult`, `original_language`, `day_of_week`, `month`, `season`, `has_homepage`;
* текстовые и составные: `overview`, `tagline`, `genre`, `companies`, `countries`, `languages`.

В рамках CP1:

* выполняется базовая проверка дубликатов;
* анализируются пропуски;
* дата релиза переводится в формат datetime;
* создаются новые признаки;
* исключаются признаки, приводящие к leakage.

## Результаты

Что сделано в CP1:

* загружен и описан датасет;
* самостоятельно определена целевая переменная;
* рассчитан ROI и построен таргет `low / medium / high`;
* выполнена первичная очистка данных;
* добавлены новые признаки;
* проведён EDA;
* выполнен `train / validation / test split`;
* обучены baseline и несколько первых моделей;
* выбрана лучшая модель по `valid_macro_f1`.

В ноутбуке сравниваются следующие модели:

* `DummyClassifier`
* `LogisticRegression`
* `RandomForestClassifier`
* `HistGradientBoostingClassifier`

По результатам CP1 промежуточным лидером стал **RandomForest**.

Результат лучшей модели на тестовой выборке:

* **Accuracy:** `0.6426`
* **Macro F1:** `0.6394`

Короткие выводы по результатам:

* baseline заметно слабее обучаемых моделей;
* в данных есть полезный сигнал для классификации;
* наиболее трудным классом оказался `medium`;
* даже без сложного тюнинга и без прямого использования текстов удалось получить рабочее качество на первом этапе.
